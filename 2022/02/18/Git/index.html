版本回退

  • –mixxed

默认选项，回退至添加前（add）前

  • –soft

回退至提交（commit）前

  • –hard

回退至修改前

  • HEAD
    • HEAD 表示当前版本
    • HEAD^ 表示上个版本
    • HEAD^^ 表示上上个版本
    • 以此类推
    • 也可以用 ~
    • HEAD~0 表示当前版本
    • HEAD~1 表示上个版本
    • HEAD~2 表示上上个版本
  • 可以指定回退某个文件，在最后加上文件名即可

.gitignore

.gitignore 文件可以让 git 忽略仓库目录下的某些文件

  • 相关语法
    • # 注释
    • *，表示任意多个字符（可以理解为所有）
    • ?，表示一个字符
    • []，方括号表示可选字符范围
    • {}，大括号表示可选字符串范围
    • !，表示例外，将不被忽略
    • /，路径分隔符，若在文件夹前，表示忽略此文件夹下的文件，而不忽略其他文件夹中的同名文件；
      若在文件夹后，表示忽略此文件夹下的子文件夹，而不是文件
